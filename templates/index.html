Архитектура и шаблоны проектирования на Python

Lesson 01 (Паттерны web-представления)

- реализация wsgi фреймворка используя паттерны page controller и front controller...
- возможность отвечать на get запросы пользователя

Lesson 02 (Архитектура python-приложений)

- возможность обработки данных из post и get запросов
- страница контактов на которой пользователь может отправить сообщение

Lesson 03 (Принципы проектирования)

- механизм наследования и включения шаблонов (базовый шаблон + несколько включенных шаблонов)
- меню, которое отображаетсят на всех страницах

Lesson 04 (Порождающие паттерны)

- реализуем сущностей Категории и Курсы по обучению, Студенты
- загрузка первичных данных из БД (SQLite), если БД нет - ее создание
- реализация интерфейса создания новых сущностей + копирования курсов + Запись студента на курс
- Logger (пишем в консоль и файл) отличает по имени при создании (Singleton)

Lesson 05 (Структурные паттерны)

- реализация альтернативного механизма URLs, по аналогии с фреймворком Flask @app(‘/some_url/’)
- декоратор @debug, для функций - в терминал выводятся название функции и время ее выполнения
- реализация неограниченной вложенности для категорий курсов (ссылается на др категорию)
- 2 новых вида wsgi-application. Первый - логирующий, второй - фейковый (на все запросы - “200 OK”, “Hello from Fake”)

Lesson 06 (Поведенческие паттерны)

- реализация CBV (Class Based Views). Возможность создавать view в виде класса (расширил CreateView: redirect_to для возврата...)
- при поступлении Студента на Курс, имитируется отправка SMS и Email (перенес из view в creation pattern, иначе при копировании или создании из БД не отрабатывают)
- API для Курсов и Студентов

Lesson 07 (Архитектурные системные паттерны)

- в БД добавил сущность 'student_course' для поддержания хранения в БД Студентов записавшихся на Курсы (сущности 'student', 'category' и 'course' были реализованы ранее)
- реализовал 'MapperRegistry' (Singletone) для автоматического (по имени сущности) выбора Класса работающего напрямую с БД
- разработал УНИВЕРСАЛЬНЫЙ 'UniversalMapper', работающий напрямую с БД (аналог ORM) (сам определяет перчень полей в передаваемых объектах, умеет работать как с обычными словарями, так и классами Моделей...) - использован паттерн 'UnitOfWork'
